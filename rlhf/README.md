# Reinforcement Learning from Human Feedback (RLHF)

This directory contains the pipeline for the RLHF part of the project: training a custom Reward Model from synthetic preference data and using it to fine-tune a PPO agent via RLHF. 

The pipeline is designed to evaluate how RLHF performance scales with the number of preference pairs ($K \in \{50, 200, 1000\}$) across continuous (`Pendulum-v1`) and discrete (`CartPole-v1`) environments.

## Repository Structure

The directory is modularized into core source code, executable pipeline scripts, and a dynamic `outputs/` folder for experiment tracking.

```text
rlhf/
├── config_rlhf.py           # Single Source of Truth for hyperparameters (e.g., KL penalty BETA)
├── reward_model.py          # Neural network architecture for scoring state-action pairs
├── rlhf_env.py              # Custom Gym Wrapper that computes rewards using the trained Reward Models and applies KL penalties
├── train_reward_model.py    # Pipeline Step 1: Trains RMs with gradient clipping (max_norm=1.0)
├── train_ppo_rlhf.py        # Pipeline Step 2: Trains PPO against the frozen RMs
├── evaluate_results.py      # Pipeline Step 3: Evaluates policies and aggregates mean/std data
├── plot_results.py          # Pipeline Step 4: Generates scaling graphs from evaluation JSON data
└── outputs/                 # Autogenerated artifact tracking
    ├── reward_models/       # Saved .pth weights
    ├── ppo_rlhf_results/    # Saved .zip PPO policies
    ├── logs/                # TensorBoard event files
    ├── evaluation_results/  # JSON files aggregating mean/std data
    └── plots/               # Final .png scaling graphs
```

The folders `ppo_rlhf_results/`, `logs/`, `evaluation_results/`, and `plots/` are further nested by the `BETA` KL penalty parameter (e.g., `beta0.1/`) to allow for clean tracking of different experimental conditions without log pollution.


## Methodology

To accurately measure sample complexity, this pipeline employs a 5-seed independent run strategy for each environment and dataset size.

* **Independent Pipelines:** It trains 5 separate Reward Models on 5 isolated datasets, then trains 5 separate PPO agents against those specific Reward Models. Our starting point model is always the Mid-Anchor policy, ensuring that all PPO agents begin with the same initial performance level.
* **Evaluation:** Final metrics are the true mathematical average (and standard deviation) of these 5 independent end-to-end runs.

## Configuration & Dynamic Tracking

Experiment tracking is handled dynamically via `config_rlhf.py`. 
To test a different KL-divergence penalty, simply change the `BETA` variable:

```python
# config_rlhf.py
BETA = 0.1
```

The pipeline will automatically create new `beta0.1/` subdirectories inside `logs/`, `ppo_rlhf_results/`, `evaluation_results/`, and `plots/`.

## Reproducing the Experiments

Execute the following 4 scripts in exact order to reproduce the 30-model experiment (2 envs × 3 K-sizes × 5 seeds). Each script corresponds to a specific phase of the RLHF pipeline, and they must be run sequentially to ensure proper data flow and artifact generation.

### 1. Train the Reward Models

Reads preference `.json` files from `data_generation/` and trains 30 PyTorch Reward Models using the Bradley-Terry loss.
```bash
python3 train_reward_model.py
```

### 2. Train the RLHF Agents
Loads the Mid-Anchor policy and fine-tunes 30 copies of it against the 30 trained Reward Models.
```bash
python3 train_ppo_rlhf.py
```

### 3. Evaluate Performance
Evaluates the Expert baselines, Mid-Anchor baselines, and all 30 PPO-RLHF agents over 50 deterministic episodes. Saves the aggregated means and standard deviations.
```bash
python3 evaluate_results.py
```

### 4. Generate Scaling Graphs
Reads the evaluation JSON and plots the True Environment Return against the Dataset Size ($K$) on a logarithmic scale, rendering performance lines and variance bands.
```bash
python3 plot_results.py
```
